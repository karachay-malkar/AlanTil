# Настройка Google Analytics 4 для AlanTil 11.8.1

## 1. Поток данных GA4

Приложение отправляет просмотры SPA вручную и использует `send_page_view: false`.

В Google Analytics открыть:

**Администратор → Сбор и изменение данных → Потоки данных → Web-поток AlanTil → Расширенная статистика → Просмотры страниц → Дополнительные настройки**

Отключить:

**Изменения страницы на основе событий истории браузера**

После настройки каждый публичный URL должен создавать ровно один `page_view`, отправленный Router.

## 2. Региональная логика

- Для России, СНГ и стран вне списка предварительного согласия статистика включается по умолчанию.
- Для ЕС/ЕЭЗ, Великобритании и Швейцарии статистика до выбора пользователя выключена.
- При отсутствии сохранённой настройки страна определяется через `src/shared/privacy/region-service.js`.
- Если страну определить не удалось, применяется безопасный режим: статистика выключена и показывается панель выбора.
- Сохранённое пользователем значение `alantil_analytics_enabled_v1` всегда имеет приоритет.

## 3. Consent Mode

До определения региона и пользовательской настройки устанавливается:

- `analytics_storage: denied`
- `ad_storage: denied`
- `ad_user_data: denied`
- `ad_personalization: denied`

При включении статистики меняется только `analytics_storage` на `granted`. Рекламные разрешения остаются выключенными.

## 6. Проверка DebugView

Открыть сайт с параметром:

`?analytics_debug=1`

Проверить:

1. Россия, новое устройство: панель отсутствует, GA4 загружается, отправляются `app_open` и один `page_view`.
2. ЕС/ЕЭЗ, новое устройство: GA4 не загружается, отображается компактная нижняя панель.
3. Кнопка «Не собирать статистику»: сохраняется `false`, панель закрывается, запросов GA4 нет.
4. Кнопка «Продолжить со статистикой»: сохраняется `true`, отправляются `app_open` и один `page_view`.
5. Ссылка «Политика конфиденциальности»: открывается раздел настройки статистики, панель остаётся до сохранения выбора.
6. Отключение в политике: устанавливается `ga-disable-G-1WSMD45Q9D`, удаляются `_ga` и `_ga_*`, новые события не отправляются.
7. Повторное включение работает без перезагрузки.
8. Back, Forward, прямые URL и Telegram BackButton работают без изменений.
9. Тексты запросов, слова, переводы и Telegram-параметры не попадают в события.
